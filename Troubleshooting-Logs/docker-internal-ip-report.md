# 도커 내부 IP (172.20.0.2) 조사 보고서 ✨

`paired.json` 파일에서 발견된 `remoteIp: 172.20.0.2` 항목에 대한 조사 결과입니다.

## 1. 개요
`paired.json` 설정 파일에 기존에 보지 못했던 `172.20.0.2`라는 IP 주소가 기록되었습니다. 해당 기기는 `clientId: cli`, `platform: linux`로 식별되었습니다.

## 2. 조사 내용
조사 결과, 해당 IP는 **OpenClaw가 실행 중인 Docker 내부 네트워크의 주소**임이 확인되었습니다.

- **IP 할당 정보**: `docker network inspect` 결과, `openclaw-gateway` 컨테이너가 Docker 브리지 네트워크에서 `172.20.0.2` IP를 할당받아 사용 중입니다.
- **기록 생성 시점**: 도커 컨테이너 내부 터미널(`docker exec` 등)에서 OpenClaw CLI 명령어를 실행했을 때, CLI 도구가 컨테이너 내부 로컬 호스트를 통해 게이트웨이에 접속하면서 해당 IP가 기록되었습니다.
- **네트워크 차이**:
    - `172.20.0.1`: Docker 네트워크의 게이트웨이(호스트 윈도우 PC)에서 접속한 경우.
    - `172.20.0.2`: Docker 컨테이너 자체 내부에서 접속한 경우.

## 3. 결론
이 기록은 외부에서 접속한 것이 아니라, **컨테이너 내부 통신**에 의해 발생한 정상적인 기록입니다. 시스템이 부팅되거나 내부 스크립트가 실행될 때 CLI 도구를 사용하면 이와 같은 기록이 남을 수 있습니다.
